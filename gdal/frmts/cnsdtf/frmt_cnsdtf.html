China Geospatial Data Transfer Grid Format (CNSDTF-DEM | CNSDTF-RAS)

(Available for GDAL >= 1.10.0)

Supported for read and write access, including reading of an affine georeferencing transform and some projections. This format is the ASCII interchange format for China Geospatial Data Transfer Grid, and takes the form of an ASCII file.

The NODATA value for the grid read is also preserved when available. 这个NODATA在标准中是没有的，此处自己添加的。

By default, the datatype returned for CNSDTF datasets by GDAL is autodetected, and set to Float32 for grid with floating point values or Int32 otherwise.

When writing floating-point values, the driver uses the "%6.20g" format pattern as a default. You can alternatively specify the number of decimal places with the DECIMAL_PRECISION creation option. For example, DECIMAL_PRECISION=3 will output numbers with 3 decimal places.

Georeferencing

The BLX projection is fixed to WGS84 and georeferencing from BLX is supported in the form of one tiepoint and pixelsize.

Creation Issues

Creation Options:

  • FORCE_RASTER=YES: Set file header data mark. Force use of RASTER (CNSDTF-RAS), default is FALSE(CNSDTF-DEM).
  • DECIMAL_PRECISION=3: You can alternatively specify the number of decimal places with the DECIMAL_PRECISION creation option. For example, DECIMAL_PRECISION=3 will output numbers with 3 decimal places.
